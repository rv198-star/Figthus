# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 画面只表达一个认知动作或核心结构。
- 出场逻辑清楚：为什么是烂笔头、破橡皮或双角色同屏。
- 至少有生成或修正中的一个动作被明确看见。
- 画面中有痕迹与空白的关系：铅笔线、擦痕、灰屑、断芯、证据单或新空白。
- 烂笔头残损可信：短、斜、脏黄、断芯风险、大包圈、残橡皮头。
- 破橡皮残损可信：粉色侧和灰白侧都破，不完整不干净，有裂缝、断面、灰痕。
- 中文标注少、短、能读，建议 3-6 个。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒、停止、裂缝或结果。
- 蓝色只用于证据、反馈或系统状态。
- 没有复刻角色设定图姿势或上游小黑旧案例构图。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 角色像儿童文具吉祥物。
- 烂笔头像全新铅笔或没有大包圈和残橡皮头。
- 破橡皮像完整干净的橡皮块，尤其粉色侧太完整。
- 双角色同屏时做同一种工作，缺少生成/修正张力。
- 单角色图里完全没有另一种力量的痕迹，导致它只是孤立角色图。
- 概念名词直接被写成画面标题，而不是转成可视动作。
- 没有可见后果：没有写痕、擦痕、空白、灰屑、断芯或证据。

## 迭代方法

- 太普通：把概念名词改成物理动作，让角色承担动作。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 deadpan、damaged、lowly、not cute、not mascot。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 出场关系不清：只保留承担核心动作的角色，并用铅笔线、擦痕、灰屑、空白、断芯或证据单补上另一种力量。
- 烂笔头太完整：加短身、掉漆、断芯、大包圈、残橡皮头。
- 破橡皮太完整：加粉色侧缺口、崩边、裂缝、灰痕、断面。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先看出当前动作由谁承担，以及另一种力量留下了什么痕迹，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上残损文具正在干笨活的认知草图，就不合格。
