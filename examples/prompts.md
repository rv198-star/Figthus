# Prompt Examples

下面这些 prompt 可以直接复制到 Codex 里使用。

## 只做配图规划

```text
Use $figthus 先不要生图。
请分析下面这篇文章哪里值得配图，输出 5 张左右的 shot list。
不要平均配图，只挑认知锚点。
每张图写清楚：
- 放在哪个段落后
- 图的主题
- 认知锚点
- 核心意思
- 可视动作
- 烂笔头在做什么
- 破橡皮在做什么
- 角色状态
- 建议元素
- 建议中文标注词

<粘贴文章>
```

## 文章正文配图

```text
Use $figthus 把下面这篇文章生成 4 张 Figthus 正文配图。
要求：16:9 横版、纯白背景、黑色手绘线稿、少量红橙蓝中文手写批注。
每张图只讲一个认知动作，不要做 PPT 信息图，不要儿童文具海报。
烂笔头负责先写/外化，破橡皮负责擦改/校准。

<粘贴文章>
```

## 长文配图策略

```text
Use $figthus 给这篇长文做配图策略。
只挑适合画的认知动作：第一版被推出去、证据被拖回来、假清楚被擦掉、失败被分类、停止规则、从空白重启。
默认 6-8 张，先输出 shot list，不要生成图片。

<粘贴文章>
```

## 按核心动作生成一组样张

```text
Use $figthus 为这篇文章生成 6 张正文配图。
优先从这些动作里选：第一笔推出去、证据拖回来、假清楚被擦掉、停止闸、从空白重启、裂开了。
每张图只讲一个动作，中文标注最多 3 个，不要左上角标题，不要 PPT 信息图。

<粘贴文章>
```

## 对齐 Figthus 成品密度

```text
Use $figthus 参考仓库里的 examples/images 样张密度，为这个观点生成 1 张图：

结果要带着证据回来，而不是让 AI 原地多试几次。

要求：白底、手绘、留白多、角色必须有分工；烂笔头推出第一版，破橡皮把证据拖回来。
```

## 单个观点生成一张图

```text
Use $figthus 为这个观点生成一张 16:9 正文配图：

先写错，才有东西可以擦近一点。

画面要怪诞但清爽。烂笔头必须制造第一条痕迹，破橡皮必须制造一块更真实的空白。
中文标注最多 5 个，短一点。
```

## 工作流主题

```text
Use $figthus 为“Loop Engineering 不是多试几次，而是让结果带着证据回来”生成一张正文配图。
不要画正式流程图，不要把 Loop Engineering 写成标题。
请重新发明一个低科技隐喻，让烂笔头推出第一版，让破橡皮拿证据判断改哪里。
```

## 改图：去掉标题

```text
Use $figthus 帮我编辑这张图。
去掉左上角的“Workflow / 流程图”标题和下划线，其他内容保持不变。
不要新增任何文字或物件。
```

## 改图：增强角色分工

```text
Use $figthus 这张图方向对，但烂笔头和破橡皮的分工不明显。
请保持核心意思不变，重生成一版：烂笔头负责制造第一版痕迹，破橡皮负责依据证据擦掉假清楚。
画面更怪一点，但仍然纯白、清爽、少字。
```

## 改图：裂开了状态

```text
Use $figthus 让这张图里的破橡皮进入“裂开了”状态。
保持构图和核心意思不变。破橡皮沿粉色和灰白断面自然裂开，不要画绳子，不要画完整橡皮块。
```
